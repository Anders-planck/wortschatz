# WortSchatz — Fase 2: Esercizi

## Overview

Add 4 types of interactive exercises: fill-in-the-blank, dictation, article/case quiz, and AI chat. Accessible from a new "Esercizi" hub in the review tab. Each exercise uses words from the user's vocabulary. Sessions track progress and provide grammar-aware feedback.

## Problem

The app teaches vocabulary through flashcard-style review (reveal/respond), but never tests the user's ability to *produce* German. There's no active recall, no grammar testing, no writing practice.

## Solution

Four exercise types, each targeting a different skill:

1. **Fill-in-the-blank (Completa)** — a sentence with a missing word. User types the answer. Tests conjugation, word form, and context.
2. **Dictation (Dettato)** — TTS plays a sentence, user writes what they hear. Tests listening + spelling.
3. **Article/Case quiz (Articoli & Casi)** — sentence with a gap for the article (der/den/dem/des). User picks from pills. Tests case governance.
4. **AI Chat** — covered separately in Fase 3 spec.

Plus a "Mix intelligente" mode where AI selects the best exercise type per word based on the user's weak points.

## Exercise Generation

All exercises are generated by Gemini AI from the user's saved words.

### Fill-in-the-blank prompt

Input: 8-12 words from vocabulary (prioritized by low review score).

Output (Zod schema): array of exercises, each with:
- `sentence`: German sentence with `___` placeholder
- `answer`: the correct word to fill in
- `translation`: Italian translation of the full sentence
- `hint`: grammar tip (e.g., "Konjunktiv II di mögen")
- `wordTerm`: which vocabulary word this tests

### Dictation sentences

Use existing example sentences from the `words` table (`examples` JSON field). TTS via existing `useSpeech` hook. No extra AI call needed — just pick sentences from vocabulary that have examples.

### Article/case quiz prompt

Input: 8-12 nouns from vocabulary with gender.

Output (Zod schema): array of exercises, each with:
- `sentence`: German sentence with `___` where the article goes
- `correctArticle`: the correct article (der/den/dem/des/die/ein/einen/einem/eines)
- `options`: 4 article options to choose from
- `caseType`: which case (nom/akk/dat/gen)
- `translation`: Italian translation
- `explanation`: why this case is used

## Screens & Navigation

### 1. Exercise hub

**Route**: `(review)/exercises.tsx` (new)

- Accessible from review dashboard via toolbar button `pencil.and.list.clipboard`
- Grid of 4 exercise types with SF Symbol icons
- "Mix intelligente" AI option at bottom
- Each card shows exercise name, icon, short description

### 2. Exercise session

**Route**: `(review)/exercise-session.tsx` (new, modal)

- Receives `exerciseType` param: "fill" | "dictation" | "cases" | "mix"
- Progress bar at top (N dots)
- Exercise content area (varies by type)
- Answer input (TextInput for fill/dictation, pills for cases)
- "Salta" / "Verifica" action bar
- Feedback banner (correct green / wrong red) with grammar tip
- "Continua" to advance

### 3. Exercise summary

Shown at end of session within the same modal:
- Score ring (% correct)
- Stats: correct / errors / time
- "Da ripassare" word chips
- "Ripeti gli errori" / "Torna" buttons

## Data Model

No new tables needed. Exercise sessions are ephemeral — generated on the fly, results affect `review_score` via existing `updateReviewScore()`. Correct answers: +1. Wrong: -1.
